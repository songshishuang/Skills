# 发布到 JoySpace（发版说明 HTML → JoySpace 文档）

把已生成的发版说明，落成一篇**格式完整、带多级目录、带截图**的 JoySpace 在线文档。本文是 release-note-builder 的「在线发布层」，依赖浏览器自动化（Chrome MCP 之类）操作 joyspace.jd.com。

> 一句话结论：**别用富文本逐字段手敲**（脆弱、慢）。走「纯净语义 HTML → 部署线上 → 全选复制 → 粘贴」一键成型，是踩了一整轮坑后的最优路径。

## 两条路径与取舍

| 路径 | 何时用 | 成本 | 可靠性 |
|---|---|---|---|
| **A 纯净 HTML 粘贴** | 默认首选。整篇创建/重灌 | 低（全选→粘贴一步） | 高（标题层级/列表/图全自动带入） |
| **B 富文本手动编辑** | 只微调几处、或没有可部署的线上 HTML | 高（逐处定位） | 低（JoySpace 富文本对坐标/选区极脆弱） |

为什么 A 远胜 B：JoySpace 富文本编辑器支持「粘贴 HTML 保留源格式」，`<h1-h4>` 自动转各级标题（生成左侧多级目录大纲）、`<b>` 加粗、`<ul/ol>` 列表、`<img>` 图片**一次性全带入**。逐字段手敲则要反复点击定位（光标频繁跑偏）、手动设标题样式（菜单脆弱）、逐张插图（剪贴板易污染）——慢且易错。

## 路径 A（推荐）：纯净 HTML 一键粘贴

### 1. 必须用「纯净语义 HTML」，不是展示用的卡片版

发版说明常有两份 HTML：
- **卡片美化版**（`templates/release-note-skeleton.html`）—— 给门户/本地展示，有卡片、配色、悬浮目录、CSS 计数器。**不能直接粘 JoySpace**。
- **纯净语义版**（`templates/joyspace-pure-skeleton.html`）—— 专供 JoySpace，只有 `h1-h4 / p / ul / ol / img / b`，编号手写进文本，**零 CSS counter / 卡片 / 悬浮目录 / ::before 伪元素**。

> **核心坑**：卡片版那些 CSS 装饰（`counter()`、`::before` 生成的编号、卡片 div、悬浮目录块）粘进 JoySpace 会被当成**文本**，变成满屏垃圾——`$` / `•$` / "目录·CONTENTS" / "PART A" / "A1"，文档标题还会被第一个文本块顶替。所以必须用纯净版。

纯净版的标题层级直接决定 JoySpace 目录结构：
- `h1` = 文档标题
- `h2` = 【上线时间】【版本内容】【功能介绍】
- `h3` = 运营端 / 供应商端（端分组）
- `h4` = `1、模型管理` 等功能（编号手写进文本，**不要**用 CSS 自动编号）

### 2. 部署到线上（图片关键前提）

图片用相对路径（如 `assets/<version>-release/02-xxx.png`）。**必须把纯净版 HTML 部署到线上**再打开——只有线上访问时相对路径才解析成线上 URL，JoySpace 粘贴才抓得到图。

> 本地 `file://` 或 `localhost` 的图，JoySpace 服务器抓不到 → 粘进去图是裂的。

部署方式随项目（如 push 到部署分支触发 CI）。部署后先 `curl` 确认线上 200（CI 有延迟，必要时轮询等生效）再继续。

### 3. 全选复制 → 粘贴

1. 浏览器打开**线上版** HTML，等图加载完。
2. 点正文 → `Cmd+A` 全选 → `Cmd+C`（剪贴板拿到富文本）。
3. 切到 JoySpace 目标文档：整篇创建就 `Cmd+A`(两次，渐进全选) + `Delete` 清空；再 `Cmd+V` 粘贴。
4. JoySpace 默认「保留源格式」，标题/加粗/列表/图全带入，左侧自动生成多级目录。
5. `get_page_text` 核对全文，确认无残留、图都在。

### 路径 A 的坑速查

- 粘出 `$` / "PART A" / "目录·CONTENTS" → 用了卡片版而非纯净版。换纯净版。
- 图是裂的 → 没开线上版（开了本地/localhost），或线上还没部署生效。
- 文档标题变成奇怪文本 → HTML 第一个元素不是 `h1`（纯净版第一个必须是文档标题 `h1`）。
- 粘错了想重来 → `Cmd+Z` 可**整篇撤销一次粘贴**，回到粘贴前。
- `get_page_text` 里列表项显示 `1.$` / `•$` → 那是工具对列表标记的提取符号，**不是文档里的字符**，正文渲染是正常的 `•` / `1.`，别误删。

## 路径 B（备选）：富文本手动编辑

只在「微调几处」或「没有可部署线上 HTML」时用。JoySpace 富文本对坐标点击/选区极脆弱，下列为踩坑后验证可靠的操作：

1. **剪贴板传图**（JoySpace 支持 `Cmd+V` 粘图）：必须用 heredoc 形式
   `osascript <<EOF` … `set the clipboard to (read (POSIX file "x.png") as «class PNGf»)` … `EOF`。
   `-e "…«class PNGf»…"` 双引号形式会把 « » 特殊字符传坏 → 粘出旧剪贴板内容。每张图前 `osascript -e 'clipboard info' | grep PNGf` 验证，防外部进程污染剪贴板。
2. **可靠光标定位**：点击该行 + `Cmd+Left`(行首) + `Cmd+Shift+Right`(选整行) + type 替换。
   ❌ triple_click 只选词（会把"版本内容"插成"版本【版本内容】内容"）；❌ 点空白光标跑偏；❌ `Cmd+Down` 会从段中间切断（把"门户工作台"切成"门"+"户工作台"乱序）。
3. **边写边插图 > 事后定位**：重写正文时 type 完描述、光标自然在行尾，紧接 `Cmd+V` 插图，定位 100% 准；事后滚动点击插图常落错位置。
4. **设标题样式（生成左侧多级目录）**：光标置于该行 → 顶部「正文▾」下拉选级别（菜单延迟约 2s 展开、toggle 式连点两次会关）。H2 章节 + H3 端 = 多级目录。
5. **JoySpace 不支持 markdown 快捷**：`## ` 不转标题、行内 `**` 不转加粗 → 加粗只能 `Cmd+B` 开关、标题只能工具栏。
6. **传图无法用 file_upload**：浏览器自动化的 file_upload 通常只接受拖进会话的附件，拒绝项目路径/下载目录 → 手动传图只能走上面的剪贴板法。

## 反向同步

JoySpace 是**富文本消费端**，HTML 是**版本库里的真相源**。如果在 JoySpace 里手改了内容（精简文案、改标题等），要把改动**回灌纯净版 HTML 源**——否则下次走路径 A 重新粘贴会覆盖掉手改。

做法：`get_page_text` 取 JoySpace 现状 → 逐项对比纯净版 HTML → python 精确替换同步 → 核对两边一致。

## JoySpace 上线公告模板范式

对齐 JoySpace 标准上线公告（参考企业内已有公告）：

```
【上线公告】<产品> VX.X · <一句话主题>        ← h1 文档标题
【上线时间】                                  ← h2
<YYYY 年 M 月 D 日>
【版本内容】                                  ← h2
<引言一句话>
1、<端> · <模块>：<扫读级摘要>                 ← 有序列表，端·模块名加粗
…
【功能介绍】                                   ← h2
<端 A>（如 运营端）                            ← h3
1、<功能名>                                   ← h4（编号手写）
<1-2 句描述>
• <要点> • <要点> • <要点>                     ← 无序列表
<截图>                                        ← img
…
<端 B>（如 供应商端）                          ← h3
…
```

要点：章节用【】包裹、功能按端分组、功能名带「N、」组内编号。这套结构让 JoySpace 左侧目录形成「章节 → 端 → 功能」多级大纲。
