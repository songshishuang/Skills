# Skills 设计计划

> 每个 Skill 落地前的设计稿——**不对外分发**，保持 `skills/{name}/` 包的干净。

## 命名约定

`{skill-name-slug}-plan.md`

例：`auto-tagging-plan.md` / `churn-analysis-plan.md`

## 标准结构（写设计计划时遵循）

```markdown
# {Skill 名} 实施计划

## 目标
这个 Skill 解决什么问题、给谁用、关键约束是什么。

## 工作流设计
1. 输入：什么样的指令 / 数据进来
2. 处理：经过哪些步骤
3. 输出：产出什么形态（artifact / 报告 / 操作）

## 关键交互
- CUI（自然语言对话）配置
- GUI（卡片审核）干预
- 异步任务执行
- 最终产出

## 评测指标
- 准确率 / 召回率 / 用户满意度

## 依赖
- 其他 Skill / 系统接口 / 数据源

## 风险
- 已知的失败场景与应对
```
