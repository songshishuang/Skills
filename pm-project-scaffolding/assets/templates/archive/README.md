# archive/

**归档层**——已废弃 / 历史快照 / 不再活跃的资产。

## 何时搬入

- 3 个月以上没人访问的目录
- 已被新版本取代的设计稿 / 文档
- 废弃的实验代码
- 大版本切换后的老版本快照（v1 升 v2 时 v1 整体归档）

## 不要轻易删除

归档可逆，删除不可逆。6 个月后再评估是否真的删除。

## 推荐子目录（按需建）

- `early-mockups/` — 早期 V1/V2 demo
- `research-{topic}/` — 已废弃的研究方向
- `legacy-{version}/` — 旧版本整体快照
- `experiments/` — 没走通的实验代码

## 沉淀准则

每个归档子目录加一份 README.md 说明：
- 归档原因
- 归档日期
- 当时的相关链接（issue / PR / 决策记录）
- 6 个月后回看建议
