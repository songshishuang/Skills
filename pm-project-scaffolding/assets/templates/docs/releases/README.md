# 上线公告 (Release Notes)

本目录沉淀**面向客户 / 最终用户**的版本发布说明。

## 与 `CHANGELOG.md` 的区别

| | `../CHANGELOG.md` | `releases/` 本目录 |
|---|---|---|
| 受众 | 内部团队 / 开发者 | 客户 / 最终用户 / 销售 / 客服 |
| 语气 | 技术变更陈述 | 营销 + 用户教育 |
| 组织 | 按版本号 | 按发布日期 |
| 写法 | "新增 X、变更 Y、移除 Z" | "现在你可以用 X 做什么，怎么用" |
| 形式 | 单文件 | 每版一份独立 markdown |

**每次发版同时更新两边**——CHANGELOG 写技术变更，本目录写客户故事。

## 命名约定

`YYYY-MM-DD-{version}-release-notes.md`

例：`2026-05-09-v1.0-release-notes.md`

## 写作模板

新增公告时复制 [`TEMPLATE.md`](./TEMPLATE.md)。

## 沉淀准则

- **截图 / 视频独立存放**：`releases/assets/{version}/`
- **客户原话上墙**：从 user-research 挑 1-2 句引用
- **数据可量化**：用"+30% 转化"而不是"显著提升"
- **保留历史版本**：不删不改已发布的，错误用新文件勘误

## 后续维护节奏

- **大版本**：必出，并附内部 changelog 索引
- **季度小版本**：累积一个季度的改进合并发一份
- **紧急补丁**：除非影响客户体验，否则不专门发——合并到下个版本
