# 用户研究 (User Research)

本目录沉淀**直接来自用户的信息**——访谈纪要、问卷数据、可用性测试报告、用户画像。

## 与 `data-analysis/` 的区别

| | 用户研究 | 数据分析 |
|---|---|---|
| 数据来源 | 直接接触用户（访谈 / 问卷 / 观察） | 系统埋点 / 日志 / 数据库 |
| 数据形态 | 定性 + 少量定量 | 定量为主 |
| 回答的问题 | "用户为什么这么做""他们在想什么" | "用户做了什么""漏斗转化率多少" |
| 典型产出 | Persona / 访谈纪要 / 用研报告 | 看板 / 分析报告 / A/B 实验结果 |

两者**互补不重叠**——用户研究告诉你"为什么"，数据分析告诉你"是什么"。

## 子目录约定（按需建）

```
user-research/
├── personas/             用户画像（Persona）
│   └── persona-{role}.md
├── interviews/           用户访谈
│   ├── YYYY-Qn-访谈记录.md
│   └── audio/            原始录音（gitignore，备份云盘）
├── surveys/              问卷
│   ├── 问卷模板-{主题}.md
│   └── results/          回收数据
└── usability-tests/      可用性测试
    └── 报告-{日期}.md
```

## 沉淀准则

- **每次访谈必有纪要**——访谈后 24 小时内写完，否则细节会丢
- **录音云盘留档但不进 git**——`.gitignore` 加 `audio/` 避免仓库膨胀
- **个人信息脱敏**——纪要里用化名（如「客户 A」「运营经理 B」）不写真实姓名 / 公司
- **季度总结报告**——每季度末把所有访谈 / 问卷合并成 trends 报告
