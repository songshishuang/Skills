# 培训手册 (Training)

> {{PROJECT_NAME}} 的全套培训材料——「我们告诉别人怎么用 / 怎么卖 / 怎么支持」。

## 与 `docs/user-research/` 的核心区别

| | docs/user-research/ | training/ |
|---|---|---|
| 信息方向 | **客户 → 我们**（我们听） | **我们 → 客户 / 团队**（我们说） |
| 内容形态 | 访谈纪要、问卷数据、画像 | 上手指南、销售话术、客服 playbook |
| 时机 | 产品规划阶段持续做 | 产品上线前必备 |

## 两大方向

### `customer/` — 面向外部客户的培训
把客户从"听说过"变成"会用"。三件套：onboarding-guide / operation-manual / faq。

### `internal/` — 面向内部团队的培训
让销售、客服、新人快速达到产出水平。三件套：sales-kit / support-playbook / new-hire-onboarding。

## 沉淀准则

- **大量截图 + 短句**——培训材料读者注意力低，用图代替长描述
- **任务驱动**——「如何完成 X」而不是「X 功能介绍」
- **跟随产品版本**——每发 Release Notes 同步更新操作手册和 FAQ
- **季度通读**——每季度末由 PM 通读一遍，确保没过时内容

## 资源管理

- 视频文件**不进 git**（`.gitignore` 排除 `*.mp4`），原始视频放云盘
- 视频**脚本进 git**，便于复用和翻译
- 截图统一放 `assets/{topic}/` 子目录
