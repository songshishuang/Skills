# 客服 Playbook

> 适用 {{PROJECT_NAME}} 版本：v0.1+
> 受众：客户成功 / 一线客服 / 售后支持
> 用途：客户使用中遇到问题时的标准应对流程

---

## 核心原则

1. **5 分钟响应** — 客户工作时间提的问题 5 分钟内必须有回应（哪怕只是「已收到，正在查」）
2. **30 分钟初判** — 30 分钟内给出问题类型分类 + 预估解决时间
3. **不绕弯** — **承认问题 + 给出明确动作 / 时间**
4. **闭环反馈** — 问题解决后主动通知客户，并把案例沉淀到 FAQ

---

## 问题分类与应对

### Class A — 使用问题（"我不知道怎么操作"）

**应对**：
1. 先看 [`../customer/faq.md`](../customer/faq.md)
2. 引导客户看 [`../customer/operation-manual.md`](../customer/operation-manual.md)
3. 文档没覆盖 → 15 分钟视频演示
4. 高频问题反馈给 PM → FAQ 升级

**响应时长**：当天

### Class B — Bug / 体验问题

**应对**：
1. 收集证据：会话 ID / 时间戳 / 期望 / 实际 / 截图
2. 内部 Bug 跟踪系统建单
3. **不要假装无事**——告诉客户预估时间
4. 修复后闭环通知

**响应时长**：
- P0（影响核心业务）：当天修复或回退
- P1（影响日常使用）：本周
- P2（小问题）：下个迭代

### Class C — 性能问题

**应对**：转 ⚠️ TODO 技术团队排查

**响应时长**：当天给方向，48 小时给阶段结论

### Class D — 数据 / 合规问题

**应对**：
1. **立即升级到 CTO / 安全负责人**
2. 暂停影响范围
3. 完整取证 + 内部根因分析
4. 24 小时内书面答复 + 整改方案

**响应时长**：立即

### Class E — 商务问题（价格 / 续约 / 增购）

**应对**：转商务团队，客服不直接答价格

**响应时长**：24 小时内商务联系客户

---

## 升级路径

```
一线客服
  ├─ 当场能答 → 答
  ├─ Class A → 文档 + 演示
  ├─ Class B → Bug 跟踪 → 工程团队
  ├─ Class C → 技术团队
  ├─ Class D → CTO / 安全（立即）
  └─ Class E → 商务团队
```

## SLA 约定（对内）

| 严重度 | 首次响应 | 阶段性进展 | 解决时间 |
|---|---|---|---|
| P0 | 5 分钟 | 30 分钟 | 当天 |
| P1 | 30 分钟 | 4 小时 | 本周 |
| P2 | 2 小时 | 1 天 | 本月 |
| P3（咨询类） | 当天 | / | 当天 |

---

## 沉淀机制

- **每月评审高频问题** — 客服 + PM 评审本月 top 10，决定哪些升级到 FAQ / 操作手册 / Release Notes
- **季度复盘 NPS** — 反馈到产品规划
- **重大问题事后复盘** — Class D 必须有 postmortem，归档到 `docs/decisions/`
