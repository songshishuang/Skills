# 新员工 30 天上手计划

> 适用所有角色（按角色挑必读章节）
> 30 天后期望状态：能独立产出本职工作 + 理解 {{PROJECT_NAME}} 产品全貌

---

## 入职第 1 周：认识产品

### Day 1-2
- [ ] 体验线上演示：⚠️ TODO URL
- [ ] 通读：[`README.md`](../../README.md)
- [ ] 通读最新 [Release Notes](../../docs/releases/)
- [ ] 体验产品至少 30 分钟

### Day 3-4
- [ ] 通读：[`docs/prd/`](../../docs/prd/) 最新 PRD
- [ ] 通读：[`docs/product-planning/`](../../docs/product-planning/)
- [ ] 通读：[`docs/roadmap.md`](../../docs/roadmap.md)

### Day 5：与 PM 1on1（必填的 5 个问题）
1. {{PROJECT_NAME}} 解决的核心客户痛点是什么？
2. 与竞品的本质区别？
3. 当前阶段的核心目标？
4. 我所在角色对这个目标的贡献？
5. 我的 30 天 OKR 是什么？

## 入职第 2 周：理解架构与技术

### 必读（所有角色）
- [ ] 通读：[`CLAUDE.md`](../../CLAUDE.md)
- [ ] 通读：[`docs/decisions/`](../../docs/decisions/)

### 工程 / 产品同学
- [ ] 跟一次完整功能开发流程

- [ ] 通读：[`sales-kit.md`](./sales-kit.md)

### 客户成功 / 客服同学
- [ ] 通读：[`support-playbook.md`](./support-playbook.md)
- [ ] 通读：[`../customer/faq.md`](../customer/faq.md)
- [ ] 跟客户群对话至少 2 小时

## 入职第 3 周：实操

### 工程同学
- [ ] 跑通本地预览
- [ ] 提交第一个 PR

### 产品 / PM 同学
- [ ] 主导一次客户访谈
- [ ] 写一份功能 PRD
- [ ] 起草一份 ADR

### 销售同学
- [ ] 跟随老销售完成 2 次客户拜访
- [ ] 独立完成 1 次客户演示

### 客户成功同学
- [ ] 独立处理至少 5 个客户问题
- [ ] 更新至少 2 条 FAQ
