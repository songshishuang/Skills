# Col-span Cookbook（12 列填满速查表）

12 列 grid 内，每行子组（subgroup-box）col-span 加起来必须 = 12。

## 单行内不同卡片数对应推荐 (col-span, inner-cols)

| 卡片数 | 推荐 col-span | inner-cols | 一行高度 | 备注 |
|---|---|---|---|---|
| 1 | 2 / 3 / 4 / 6 / 12 | 1 | 1 行 | 按邻居子组占用空间反推 |
| 2 | 4 / 6 / 8 / 12 | 2 (横排) | 1 行 | 卡片并排 |
| 2 | 3 / 4 / 6 | 1 (堆叠) | 2 行 | 卡片纵堆叠（卡片宽度足够时） |
| 3 | 6 / 12 | 3 | 1 行 | 3 卡横排 |
| 3 | 3 / 4 | 1 | 3 行 | 3 卡纵堆叠 |
| 4 | 4 / 8 / 12 | 2 | 2 行 | 2×2 |
| 4 | 6 / 12 | 4 | 1 行 | 1×4 横排 |
| 5-7 | 12 | 4-5 | 2 行 | 多于 4 卡时建议占整行 |
| 8 | 12 | 4 | 2 行 | 1×4 + 1×4（如 8 节点） |
| 9-12 | 12 | 4-6 | 多行 | inner 取 4-6 |

## 经典组合模板

### A. 4 子组完美填满（每组 1-2 卡）

```
sub3-A col-3 + sub3-B col-3 + sub3-C col-3 + sub3-D col-3 = 12
```

适用：每个子组 1 卡 inner-1（D 域 4 个监控告警 / 应急 / 全局视图等场景）

### B. 3 子组均分（每组 2 卡）

```
sub3-A col-4 inner-2 + sub3-B col-4 inner-2 + sub3-C col-4 inner-2 = 12
```

适用：A 域 6 卡分 3 个 sub3 各 2 卡

### C. 大子组占主 + 小子组填边

```
sub3-A col-6 inner-2 (主) + sub3-B col-3 inner-1 + sub3-C col-3 inner-1 = 12
```

适用：B 域 4 + 1 + 1 = 6 卡分布

### D. 整行占满（卡片多）

```
sub3-A col-12 inner-4  (8 卡 = 2 行)
```

适用：B2 评测节点 8 个 / B3 评测集 7 个等

### E. 大整行 + 一行 3 等分

```
行 1: sub3-A col-12 inner-4 (8 卡 = 2 行)
行 2: sub3-B col-4 inner-2 + sub3-C col-4 inner-1 + sub3-D col-4 inner-3 = 12
```

适用：L2 服务层 B 域（节点 8 + 调度 2 + 产出 3 + 运维 3）

### F. 整行 1 卡（避免堆叠尴尬）

```
sub3-A col-12 inner-1
```

适用：某子组只有 1 卡时，让它占整行而非 col-2 留 col-10 空白；同时配合下方多卡子组形成节奏感

## 高度对齐技巧

同一行内的多个子组高度差 > 2 倍时（如 col-3 一卡 1 行 vs col-3 八卡 8 行），视觉上很难看，且短子组下方有大片空白。

**对策**：
- 把多卡子组拆到下一行，独占 col-12
- 或调整 col-span：让多卡子组占更宽 col，inner 增加，行数减少

例：
```
错: col-3 (1卡 1行) + col-9 (8卡 inner-1 = 8行)  ← 行高差 8 倍
对: col-4 (1卡 1行) + col-8 (8卡 inner-4 = 2行)  ← 行高差 2 倍
```

## 常见错误模式

### ❌ 固定 col-span-3 等宽分配

```
行 1: 子组 A (1卡) col-3 + 子组 B (2卡) col-3 + 子组 C (1卡) col-3 + 子组 D (1卡) col-3
高度: 1 / 2 / 1 / 1 行 → 子组 A/C/D 下方留 1 行空白
```

应改为按内容动态：col-2 + col-6 + col-2 + col-2

### ❌ 多卡子组 inner-1 堆叠产生过长一列

```
行 1: 子组 A (8卡) col-3 inner-1
高度: 8 行；同行其它子组下方一片空白
```

应改为 col-12 inner-4（独占行）或 col-8 inner-3。

## 决策树

```
当前子组有 N 卡：
├── N == 1 → col-span 看同行其他子组占用，inner-1
├── N == 2 → col-span-3/4/6, inner-1 或 inner-2（取决于卡片宽度）
├── N == 3 → col-span-3/4/6/12, 1 行或纵堆叠
├── N == 4 → col-span-4/8/12, inner-2 或 inner-4
├── N == 5-7 → 强烈建议 col-span-12 inner-4 多行
└── N >= 8 → 必须 col-span-12 inner-4 多行
```

## 验证脚本

写完每一行后，把 col-span 数字加起来：

```python
row_1 = [3, 9]      # sum = 12 ✓
row_2 = [3, 3, 3, 3]  # sum = 12 ✓
row_3 = [6, 6]      # sum = 12 ✓
row_4 = [4, 4, 4]   # sum = 12 ✓
row_5 = [2, 4, 3, 3]  # sum = 12 ✓
row_6 = [8, 4]      # sum = 12 ✓
row_7 = [12]        # sum = 12 ✓
```

加起来不等于 12 → 必有空白或溢出。
