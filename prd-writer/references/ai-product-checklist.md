# AI 产品 PRD 特化 Checklist

> 普通产品 PRD 之外，AI 产品必须额外覆盖的 7 个维度。每个新版本评审前对照检查。

---

## 一、模型层

- [ ] **主模型选择**：哪个模型？为什么选它（成本 / 时延 / 能力）？
- [ ] **Fallback 链**：主模型挂了用哪个？兜底链路打通吗？
- [ ] **版本固化**：主模型版本号写在 PRD 里（如 `gpt-4o-mini-2024-07-18`），避免静默升级
- [ ] **成本预估**：日均调用量 × 单次 token × 单价 = 月度预算
- [ ] **限流策略**：QPS 超限怎么办（队列 / 降级 / 拒绝）

## 二、提示词层

- [ ] **System Prompt 版本管理**：在 selfevo_app「系统提示词」模块管理，可一键回滚
- [ ] **变量注入**：哪些占位符必填？少了会报错还是降级？
- [ ] **Few-shot 示例**：是否需要？哪些 case 必须作为示例？
- [ ] **指令冲突检测**：业务规则之间互相冲突如何识别和处理
- [ ] **越权指令防御**：用户输入"忘记前面规则" / "执行系统命令" 怎么挡
- [ ] **生成式 AI 备案**：国内服务必须办理网信办备案，备案号是否已拿到？什么时候到期？
- [ ] **算法备案**：算法推荐 / 深度合成是否需要算法备案？

## 三、知识库 / 检索层（RAG 必填）

- [ ] **知识库范围**：哪些 KB 参与召回？每个 KB 的 owner 是谁？
- [ ] **召回策略**：BM25 / 向量 / 混合？top-K 多少？
- [ ] **Rerank 策略**：是否需要 rerank model？
- [ ] **冷启动**：上线时 KB 是空的怎么办（兜底回复 / 强制人工 / 引导补 KB）
- [ ] **更新机制**：KB 更新后多久能在模型回答里生效？需要重建索引吗？
- [ ] **过期处理**：政策类信息（如国补）过期了怎么自动下线

## 四、工具调用 / Agent 层（Function Calling）

- [ ] **工具白名单**：本场景可调用哪些 tool？严禁调用哪些？
- [ ] **工具参数 schema**：每个工具的入参 / 出参定义清晰
- [ ] **工具失败处理**：tool 调用超时 / 报错时模型怎么响应
- [ ] **工具调用次数上限**：防止死循环
- [ ] **越权工具调用监控**：调用了不该调的 tool 时如何告警

## 五、效果评估层

- [ ] **离线评测集**：Golden / BadCase / 对抗集，分别多少条？
- [ ] **评测维度**：准确性 / 相关性 / 召回 / 安全 / 时延，每个有量化门槛
- [ ] **LLM-as-judge 偏差**：用谁评？跟主模型不同的好（避免自吹自擂）
- [ ] **人工抽检比例**：自动评测之外，10% 人工 sample 校准
- [ ] **回归门槛**：新版本必须不退步（除非有理由）

## 六、上线 / 灰度层

- [ ] **灰度颗粒度**：项目级 / 客服级 / 用户级 / 流量比例？
- [ ] **灰度准出标准**：什么指标达到什么值才能推全量？
- [ ] **A/B 实验**：是否需要 A/B？怎么分流？显著性怎么判？
- [ ] **回滚速度**：从决定回滚到完成需要多久（理想 < 5 min）
- [ ] **回滚粒度**：能不能只回滚某个 skill？还是必须整体回滚？

## 七、可观测性层

- [ ] **Trace 链路**：每次 LLM 调用都有 trace_id，能从前端按 ID 反查到 token / prompt / output
- [ ] **关键字段**：trace_id / model / prompt_version / latency_ms / token_in / token_out / confidence / tool_calls
- [ ] **采样存储**：100% 调用都存太贵，按规则采样（如 失败必存 / 兜底必存 / 1% 成功采样）
- [ ] **告警阈值**：错误率 / 超时率 / 兜底率，分别多少触发告警
- [ ] **自进化闭环**：失败 case 能不能自动进入 BadCase 池供归因
- [ ] **审计日志**：技能配置 / 提示词修改全留痕，谁、什么时候、改了什么

---

## 8 个常见漏写项

按踩坑频率排序：

1. **冷启动场景**：KB 空 / 用户首次使用，AI 回什么？
2. **多技能冲突**：用户同时触发 2 个技能时怎么选
3. **会话上下文长度**：超出 context window 怎么截断 / 摘要
4. **图片 / 音频输入**：当前模型不支持时优雅降级到「请用文字描述」
5. **PII 脱敏**：手机 / 身份证 / 银行卡 → 上传给模型前要不要脱敏
6. **违禁词过滤**：模型输出后做不做最终过滤？
7. **多轮对话状态**：用户改主意时如何更新已收集的信息
8. **价格 / 库存类信息**：禁止模型生成，必须来自工具调用

## 评审会必问的 5 个问题

1. **"如果模型挂了，用户体验是什么样？"** — 检验兜底
2. **"上线后怎么知道是变好了还是变坏了？"** — 检验指标
3. **"我们能多快回滚？"** — 检验灰度+回滚
4. **"failure case 怎么进入下一版优化？"** — 检验自进化闭环
5. **"竞品/baseline 跑同样的 case 是什么效果？"** — 检验「skill 真的有用吗」
